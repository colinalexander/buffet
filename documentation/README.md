# Documentation Hierarchy

This folder defines the governing documentation for the repository.

## Authority Order (Highest to Lowest)

1) `documentation/JUDGMENT_AUTHORITY.md`
2) `documentation/PROCEDURE_CONTRACT.md`
3) `documentation/JUDGMENT_RECORD_SCHEMA.md`
4) `mandates/**/mandate.yaml`
5) `judgment_loops/**/procedure.md`
6) `src/buffet/**`

## Interpretation Rule

If any lower-level artifact conflicts with a higher-level authority, the
conflict is invalid by definition and requires escalation.
